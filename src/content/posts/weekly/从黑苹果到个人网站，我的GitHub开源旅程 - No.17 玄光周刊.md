## 我的开源旅程

> 我的 GitHub 主页：[https://github.com/cgartlab ](https://github.com/cgartlab)

我最早接触到的开源这个概念是在疫情初期，当时被关在家里百无聊赖，于是开始折腾起了黑苹果这个东西。所谓黑苹果就是在 Windows PC 上安装苹果系统。那个时候苹果还没有出 M 系列芯片，正版机器普遍都存在高价低配的问题，所以为了能同时用 Windows 和 Mac，开始了折腾。

当时国内讨论黑苹果的论坛很少，网站就那么几个。折腾过程不表，和开源有关的就是想让苹果系统认为一台 Windows PC 是正常的苹果硬件，就需要对系统的引导文件进行一定的参数调整。包括不限于，各种驱动，硬件 ID 号，型号伪装等各种代码增减。

那时候我只是一个“索取者”——在论坛和 GitHub 上找到别人分享好的 EFI 文件夹，下载、套用，能成功开机就算胜利。我很少去想这些文件是谁写的，又为什么要免费分享出来。

直到有一次，我遇到了一个毫无头绪的黑屏问题，只好硬着头皮去翻那些驱动和补丁的源码注释。在那些混乱的英文文档和代码提交记录里，我看到了几十上百人在为一个硬件的 ID 号反复调试、讨论。**那一刻我突然意识到：我电脑里运行的黑苹果系统，不是某个公司施舍的残次品，而是一群志愿者用业余时间，一砖一瓦搭建起来的。**

那种感觉和单纯下载一个破解软件完全不同。我虽然只是改了改参数，但已经能感受到自己站在一个巨大的合作网络里。原来有那么多人遇到过和我一样的问题，这也让我开始思考：我能不能也从“拿来”变成“参与”？

解决了大大小小十几个问题后，系统渐渐开始稳定下来。甚至，我开始把它用作搞定商单的主力机。稳定使用一段时间后，也尝试[把我自己的引导文件上传到GitHub上了](https://github.com/cgartlab/Hackintosh---MSI-B360m-Mortar)。
